# The Deception Layer

**Status: [DESIGN] stub.**

D3FEND Deceive is the cheapest high-fidelity detection available and it is native to counter-adversary work. This file will develop three deployments, each near-zero false positive:

- Canary documents seeded in the RAG index that no legitimate query should surface. Retrieval of one is a high-fidelity injection or scope-breach signal.
- Decoy credentials or a decoy tool present in some bind manifests that no legitimate task uses. Invocation means the agent is being driven.
- A honeypot MCP server on the allowlist that does nothing real. Any call is an injected or rogue agent.

Why it matters here: deception turns the statistical layer's noise into a clean signal, because a deception asset has no legitimate reason to be touched. An alert is an adversary, not a false positive.
